GDB Command Reference
The GNU debugger (GDB) commands facilitate debugging. The most commonly used commands are listed below. For a complete GDB command list, refer to the "GNU Debugger" chapter of the Embedded System Tools Reference Manual.
The following table briefly describes the commonly used mb-gdb or powerpc-eabi-gdb console commands. The equivalent interface versions can be easily identified in the mb-gdb or powerpc-eabi-gdb window icons. Some of the commands, such as info target and monitor info, might only be available in the console mode.
CommandDescription
load [program]Load the program into the target.
b mainSet a breakpoint in function main.
cContinue after a breakpoint.
lView a listing of the program.
nStep one line (stepping over function calls).
sStep one line (stepping into function calls).
stepiStep one assembly line.
info reg  View register values.
p xyzPrint the value of xyz data.
hbreak mainSet Hardware breakpoint in function main.
watch gvar1Set Watchpoint on Global Variable gvar1.
rwatch gvar1Set Read Watchpoint on Global Variable gvar1.
Refer to the following for more information about using GDB commands:
  •  In XPS, to open the Console Window at the bottom of the XPS Main Window, select View > Console Window. In the GDB console window, you can use all advanced GDB commands.
  •  For comprehensive online documentation on using GDB, refer to the GNU website: http://sources.redhat.com/gdb/documentation.
  •  For information about the GDB Insight interface, refer to the Red Hat Insight website: http://sources.redhat.com/insight.
