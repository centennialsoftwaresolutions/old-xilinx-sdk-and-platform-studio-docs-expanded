Using SDK Batch Mode

SDK Help Commands

SDK help commands are different from the Hardware Software Interface (HSI) help commands.

All SDK help commands are formatted as help_<command name>, such as help_create_project.

Help for the commands create_project , build , and clean can also work with the format <command name> -help , such as create_project -help.

All of the HSI commands are exported through SDK batch mode. HSI commands don't consider workspace. For usage details refer to the Generating Basic Software Platforms Reference Guide (UG1138).

SDK batch mode commands are listed in SDK Batch Mode Commands.

How it Works

The SDK batch mode utility is based on Tcl and is delivered in the SDK install.

To start SDK batch mode, use the command xsdk -batch .

Once SDK is started in batch mode, it provides the Tcl prompt and you can use all of the supported commands.

How to Run Tcl Script Files

You can run your Tcl script file using the xsdk -batch -source test.tcl command.

Example test.tcl

#!/usr/bin/tclsh

set_workspace /tmp/test1

create_project -type hw -name hw_0 -hwspec system.hdf

create_project -type bsp -name bsp_0 -hwproject hw_0 -proc microblaze_0 -os standalone

create_project -type app -name helloworld -hwproject hw_0 -proc microblaze_0 -os standalone -lang C -app {Hello World} -bsp bsp_0

build -type bsp bsp_0

build -type app helloworld

clean -type bsp bsp_0

clean -type all

build -type all

exit

User Flow

The following user flow describes how you can use the SDK batch mode.

  1. Start sdk batch mode using the command:

xsdk -batch .

  1. Set the workspace (where you want to create SDK projects) using the command:

set_workspace <Workspace Path>

  1. If you have any local repositories, set the user repository path using the command:

set_user_repo_path <Repository Path>

  1. Create a hardware project using the command:

create_project -type hw -name <hw project> -hwspec <.hdf/.xml path>

  1. Create a BSP for the hardware project using the command:

create_project -type bsp -name <bsp project name> -hwproject <hw project> -proc <processor> -os <OS Name>

  1. Create an application project using the command:

create_project -type app -name <Application Name> -hwproject <hw Project> -proc <processor> -os <Os Name> -lang <Language> -app <Application Template> -bsp <Bsp Project>

  1. Build the projects using the command:

build

  1. Clean the projects using the command:

clean

You can also open the projects created in SDK GUI mode with the command:

xsdk -workspace <workspace path>
