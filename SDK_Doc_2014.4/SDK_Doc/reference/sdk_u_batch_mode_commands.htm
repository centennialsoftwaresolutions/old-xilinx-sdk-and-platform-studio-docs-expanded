SDK Batch Mode Commands

SDK batch mode supports all the commands supported by the Hardware Software Interface (HSI), which are listed in the Generating Basic Software Platforms Reference Guide (UG1138). It also supports the following commands.

set_workspace

Description

You can use this to set your workspace.

Usage

set_workspace <Workspace Path>

get_workspace

Description

Display current workspace path.

Usage

get_workspace

set_user_repo_path

Description

Use this command to set your repository. This repository path will be used when you run the create_project command.

Usage

set_user_repo_path <Repository Path>

get_user_repo_path

Description

Display the currently set repository path.

Usage

get_user_repo_path

create_project

Description

You can use this command to create the hardware platform project, Board Support Package (BSP) project, and application project using this command.

Make sure that you set the workspace before using this command.

When creating a BSP, specify the hardware platform project available in the workspace.

When creating an application project, specify the hardware platform and bsp projects that should be available in the workspace.

For full information on this command, use create_project -help .

Usage

create_project [options]

Create a hw/bsp/application project

create_project -type hw -name <HW Project Name> -hwspec <Hw Design path .xml/.hddf>

Create a Hardware (hw) Project.

create_project -type bsp -name <BSP Project Name> -hwproject < HW Project Name> -proc <Processor> -os <OS>

Create a BSP Project.

create_project -type app -name <App Project Name> -hwproject < HW Project Name> -proc <Processor> -os <OS> -lang <Language C/C++> -app <Application Template> -bsp <BSP Project Name>

Create an Application Project for bare-meta/standalone OS usage

create_project -type app -name <App Project Name> -hwproject < HW Project Name> -proc <Processor> -os <OS> -lang <Language C/C++> -app <Application Template>

Create an Application Project for Linux OS usage

Options

-type <project-type>

Project type can be hw , bsp , or app .

Default type is hw .

-name <project-name>

Project name that should be created.

-app <template-application-name>

Name of the template application.

Default is "Hello World".

This option is valid only for creating application projects.

-proc <processor-name>

Processor instance to use for creating application/bsp project.

This option is valid only for creating application/bsp projects.

-hwspec <HW specification file>

HW specification file for creating a hardware project.

This option is valid only for creating hardware projects.

-hwproject <hw project name>

HW project for which the application or bsp project should be created.

This option is valid only for creating application/bsp projects.

-bsp <BSP project name>

BSP project for which the application project should be created.

This option is valid only for creating application projects.

If this option is not specified, a default bsp is created.

-os <OS name>

OS type for the application project.

Default type is standalone .

-lang <programming language>

Programming language can be c or c++ .

-help

Print the help for the command.
build

Description

Build projects available in the workspace.

Usage

build [options]

Build a bsp/application project or all projects in the workspace.

Options

-type <project-type>

Project type can be all , bsp , or app .

Default type is all .

-name <project-name>

Project name to be built.

-help

Print the help for the command.

 
clean

Description

Clean projects available in workspace.

Usage

clean [options]

Clean a bsp/application project or all projects in workspace.

Options

-type <project-type>

Project type can be all , bsp , or app .

Default type is all .

-name <project-name>

Project name that should be cleaned

-help

Print the help for the command.
