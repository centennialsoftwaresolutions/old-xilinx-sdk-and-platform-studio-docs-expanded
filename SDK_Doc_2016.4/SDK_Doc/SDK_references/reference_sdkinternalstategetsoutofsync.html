SDK internal state gets out of sync with actual target status if Zynq board is switched off during execution of an application

Issue

If the Zynq-7000 AP SoC board is switched off during execution of an application, SDK does not get the power-off notification and does not update its status about the target. This causes any consequent operations on the board to give errors.

Solution

Restart the XMD session from the XMD console view or restart SDK.
