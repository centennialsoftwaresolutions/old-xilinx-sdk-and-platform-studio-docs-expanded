Debugging an OpenCV Application

To run an OpenCV application on a target:

  1. Copy the ZC702 Petalinux image in to the SD card and boot from the SD card.
  2. After a successful boot, obtain the board IP address.
  3. Select Linux TCF Agent from the Target Connections view.
  4. Double-click Linux Agent.
    The Target Connection Details dialog box appears.
  5. Specify the board IP address, obtained during the boot, in the Target Connection Details dialog box.
  6. Create a new folder /tmp/opencv_lib in the Linux file system.
  7. Copy the Opencv libraries to the newly created /tmp/opencv_lib folder in remote file system.
    Note: The OpenCV libraries are located in the \<install_location>\SDK\<version>\data\embeddedsw\ThirdParty\opencv folder of your SDK installation. Where <install-location> is the SDK installation file path and <version> is the current version of the SDK.
  8. Create a Linux OpenCV application. For more details, refer Creating an OpenCV Template Application.
  9. Build the application.
  10. Select Run > Debug Configurations.
    The Debug Configurations dialog box appears.
  11. Click on Xilinx C/C++ application (System debugger) to create a new debug configuration.
  12. On the Target Setup tab page, select Linux Application Debug from the Debug Type list.
  13. On the Application tab page, specify the local .elf file path and the remote .elf file path.
    Note: The remote .elf file path is /tmp/opencv.elf.


  14. On the Environment tab page, click on the New button.
    The New Environment Variable dialog box appears.
  15. Set /tmp/opencv_lib file path to the OpenCV libraries in the LD_LIBRARY_PATH environment variable .


  16. Click OK to create the environment variable and close the New Environment Variable dialog box.
  17. Click Debug.
    You will be able to debug the OpenCV application using the system debugger.
