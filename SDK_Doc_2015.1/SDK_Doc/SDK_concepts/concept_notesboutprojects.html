A Few Notes About Projects

Note the following about projects:

  • When you create a file within a project, a record of every change is created, called the Local History. For more information about local history, refer to Workbench User Guide > Reference > User interface information > Development environment > Local history in the Help contents.
  • Spaces in projects and filenames can cause problems with some tools, such as the make utility and the compiler.
  • Be careful when you use only capitalization to distinguish files and projects. UNIX-based operating system file names are case sensitive, but Windows filenames are not. Therefore, Hello.c and hello.c are separate files in UNIX but overwrite each other in Windows.

For more information about projects, select Workbench User Guide > Concepts > Workbench > Resources.
