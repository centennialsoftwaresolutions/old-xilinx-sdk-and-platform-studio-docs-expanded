JTAG Settings

SDK communicates to the hardware over the JTAG interface for debugging, running, and profiling an application and for programming FPGA and Flash. You are required to use the Xilinx® JTAG cable or third-party cables that provide Xilinx® cable plug-in for communication. The following settings are required for JTAG communication.

  • JTAG Cable - Xilinx provides different JTAG cable types such as Xilinx Parallel Cable, Xilinx USB Cable, and Digilent USB Cable. You can connect multiple cables to your host machine. Use this option to select the JTAG cable to use for communication and customize the configurations. The default type of Auto Detect is recommended when only one cable is connected to your host machine.
  • Xilinx System Debugger Cable - The cable used when communicating over hw_server. The addtional parameters for this cable include the host machine name on which the hw_server is launched and the related port number.
  • 3rd Party Cable - You can use a third party cable manufactured by Xilinx cable partners who provide cable plug-in to integrate with SDK. A list of supported cable partners is available on the Xilinx website.
  • Board JTAG Device Chain Definition - The hardware board can have a series of JTAG devices chained together. SDK requires the JTAG Instruction Register (IR) length , IDCODE , and position in the chain of each JTAG device for communication. SDK maintains a list of these values for Xilinx devices, but you must provide the JTAG information for non-Xilinx devices present on the board.
