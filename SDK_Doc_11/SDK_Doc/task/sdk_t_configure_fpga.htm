Programming the FPGA

To program the FPGA device with the bitstream, do the following:
  1. Select Tools > Program FPGA.
  2. In the Bit File and Bmm File fields, specify the bitstream (.bit) and Block RAM Memory Map (.bmm) files to use for programming the FPGA. If these files are provided as part of the XPS exported files, SDK will automatically pick them up and populate these fields.
  3. SDK automatically detects the processors in the system and shows them in a table at the bottom of the window. In the Initialization ELF column, select the executable (.elf) file to initialize at the reset start address for each processor. This is the program the processor will start executing when it comes out of reset. From the drop-down list, you can select BootLoop or Browse to any other ELF file.
  4. Click Save and Program.

SDK programs the FPGA over the JTAG cable using the settings you specified. Depending on the size of the FPGA device and the JTAG cable frequency, programming the FPGA could take longer than expected.



Initializing XMDStub to FPGA

To debug a software application using XMDStub monitor on MicroBlaze, you should initialize the FPGA with the XMDStub monitor. The XMDStub monitor is generated for each software platform when you specify an xmdstub_peripheral instance to use in the software platform settings. For information about XMDStub, refer to Debug overview.

To locate the XMDStub monitor, do the following:

  1. Select Browse from the Initialization ELF drop-down list.
  2. Browse to software platform directory of the application to locate the XMDStub monitor - <Software Platform Name>/<Processor Instance Name>/code/xmdstub.elf.
