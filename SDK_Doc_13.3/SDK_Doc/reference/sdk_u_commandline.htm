Xilinx SDK Command Line Flows

The majority of the flows within SDK are interactive. Most of these flows are not available, neither do they make sense for command line users. However, almost all software development teams expect to be able to build executables from sources (often present under version control) in a command line flow without invoking the GUI. The following sections explain the different Eclipse and SDK entry points for such use cases.

Building applications

This section explains how users can build an application from the command line. It is assume that SDK projects are stored in a version control system. The steps below show how to import those projects into a new workspace and then build the executables.

Assumptions

The variables used in the steps below are as follows:

  • XILINX_EDK - environment variable that points to the EDK or SDK installation.
  • XILINX - environment variable that points to the XILINX ISE installation.
  • ECLIPSE - a shortcut to the Eclipse console application inside SDK installation: $(XILINX_EDK)/eclipse/lin/eclipse/eclipse, or $(XILINX_EDK)/eclipse/lin/eclipse/eclipsec.exe depending on the platform.
  • WSPACE - path to the workspace to be used.
  • VM - Path to Java VM used by SDK. This should be: $(XILINX)/java6/lin/jre/bin
The steps also reference the following three Eclipse/SDK projects, all of which are typically stored in the same folder.
  • HW - path to SDK's hardware platform project.
  • BSP - path to SDK's BSP project
  • APP - path to SDK C/C++ application project.

Import Projects into the workspace

To import the 3 projects into the workspace, issue the following command from a shell:

$(ECLIPSE) -vm $(VM) -nosplash -application org.eclipse.cdt.managedbuilder.core.headlessbuild \
   -import $(HW) \
   -import $(BSP) \
   -import $(APP) \
   -data $(WSPACE) \
   -vmargs -Dorg.eclipse.cdt.core.console=org.eclipse.cdt.core.systemConsole

Note that if your workspace already has these 3 projects, then this step can be skipped.

Build Imported Projects

To compile the BSP and the applicaiton, issue the following commands from the shell:

$(ECLIPSE) -vm $(VM) -nosplash -application org.eclipse.cdt.managedbuilder.core.headlessbuild \
   -build $(BSP) \
   -data $(WSPACE) \
   -vmargs -Dorg.eclipse.cdt.core.console=org.eclipse.cdt.core.systemConsole

$(ECLIPSE) -vm $(VM) -nosplash -application org.eclipse.cdt.managedbuilder.core.headlessbuild \
   -build $(APP) \
   -data $(WSPACE) \
   -vmargs -Dorg.eclipse.cdt.core.console=org.eclipse.cdt.core.systemConsole

Adding Software Repositories to a Workspace

If your BSP depends on drivers or libraries present in a software repository, then it is necessary to add the repository to the workspace before attempting to compile the projects. This can be performed using the following command:

$(ECLIPSE) -vm $(VM) -nosplash -application com.xilinx.sdk.sw.AddSwRepositoryApp "path/to/repo" \
   -data $(WSPACE)
