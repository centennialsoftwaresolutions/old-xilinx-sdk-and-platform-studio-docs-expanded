  • Case 1 & Case 3: Steps: 1 to 8
  • Case 2 & Case 4: Steps: 1 to 7 and 9
  • Case 5 & Case 6: Steps: 1 to 6 and 10

  1. Start Project Navigator.
  2. Select File >New Project.
    The New Project dialog box opens.
    1. Enter the project name, location, and select HDL in the Top-level source type drop-down list, then click Next to move to the Device Properties page.
    2. Select the product, category, family, device, package, and speed options. Select Verilog for cases 1, 3, 5 and 7. Select VHDL for cases 2, 4, 5, and 8. Click Next to proceed to the Project Summary window.
    3. Click Finish.
      Project Navigator creates a new project from information in the Project Summary page.
    4. Right-click the device type in the new project and select New Source.
      The New Source wizard opens.
    5. Select Embedded Processor, enter a file name, directory path location, and click Next.
      The Summary page opens.
    6. Click Finish.
      Project Navigator starts Platform Studio and prompts you to create a base system using the BSB Wizard.
  3. Click Yes.
    The Base System Builder starts. Follow the on-screen instructions to create a new EDK design.
  4. Select the I would like to create a new design radio button, then click Next.
    1. In the Board Selection page, select the board you would like to use, then click Next to proceed to the System Configuration page.
      Note The wizard only displays boards which have devices that match the one you previously selected in Project Navigator.
    2. Choose the processor, either single or dual, then click Next to proceed to the Processor Configuration page.
    3. In the Processor Configuration page, choose the processor configuration options you want to use, then click Next to proceed to the Peripheral Configuration page.
    4. In the Peripheral Configuration page, select your I/O interfaces and click Next.
      The Cache Configuration page opens. Click Next.
    5. In the Application Configuration page, select your sample C applications and the location in memory where they will be located, then click Next.
    6. In the Summary page, generate the system by clicking Finish.
  5. Exit Platform Studio.
  6. If ModelSim is selected then compile libraries using Compxlib.
  7. You now see that the EDK design is part of your sources in Project Navigator. Create an Instantiation Template by highlighting the XMP file and double-clicking on View HDL Instantiation Template in the Processes window.
    When complete, a file called system_xps.tfi is created and displayed.
    Contents of system.xps.tfi.vhi:
    • For Verilog:
      // Instantiate the module

(* BOX_TYPE = "user_black_box" *)
 
system_xps instance_name (
     
    .fpga_0_clk_1_sys_clk_pin(fpga_0_clk_1_sys_clk_pin),
      
    .fpga_0_rst_1_sys_rst_pin(fpga_0_rst_1_sys_rst_pin)  
   
    );
    • For VHDL
      -- VHDL Instantiation Created from source file system_xps.vhd

-- Notes:  

-- 1) This instantiation template has been automatically generated using types

       fpga_0_rst_1_sys_rst_pin : IN std_logic;);

                         						fpga_0_rst_1_sys_rst_pin =>);
  8. This step is specific to cases 1 and 3.
    1. Save the instantiation template file by clicking on File > Save As. Save the file using the name system_pn_top.v.
      Note It is important to save this file with another name because it is overwritten on future runs of the tools. If you make changes to the files, they might accidentally get lost.
    2. Highlight the system_pn file, right-click and select Add Source. Search for the file system_pn_top.v in your project directory and click Open. Associate it with All, and click OK.
    3. In the Sources window, select Behavioral or Post-Route simulation from the drop-down menu.
    4. Edit the system_pn_top.v file. If it is in a non-editable state, close it and reopen it by double-clicking on the file in the Sources window. Modify the code in the system_pn_top.v file as shown below and save the file.
      
module system_pn_top(fpga_0_clk_1_sys_clk_pin,  fpga_0_rst_1_sys_rst_pin);

input fpga_0_clk_1_sys_clk_pin;

input fpga_0_rst_1_sys_rst_pin;

// Instantiate the module

(* BOX_TYPE = "user_black_box" *)

system_xps 
system_xps(.fpga_0_clk_1_sys_clk_pin(fpga_0_clk_1_sys_clk_pin),
 
         .fpga_0_rst_1_sys_rst_pin(fpga_0_rst_1_sys_rst_pin));
	 
endmodule
    5. Highlight the system_pn in the Hierarchy window, right-click and select New Source. Select source type as Verilog Test Fixture and specify file name tb. Click Next. Associate the new source to system_pn_top and click Next. Click Finish in the summary window.
    6. Edit the tb.v file.
      module tb;

// Inputs

reg fpga_0_clk_1_sys_clk_pin;

reg fpga_0_rst_1_sys_rst_pin;
system_pn_top system_pn_top (

               .fpga_0_clk_1_sys_clk_pin(fpga_0_clk_1_sys_clk_pin), 
               .fpga_0_rst_1_sys_rst_pin(fpga_0_rst_1_sys_rst_pin)
               );	
endmodule
    7. While selecting the tb.vhd file in the Sources window, expand the simulator tree in the Processes window and double-click Simulate Behavioral Model or Post-Place & Route Model.
  9. This step is specific to cases 2 and 4.
    1. Save the instantiation template file by clicking on File > Save. Save the file using the name system_pn_top.vhd.
      Note It is important to save this file with another name because it is overwritten on future runs of the tools. If you make changes to the files, they might accidentally get lost.
    2. Highlight the system_pn in the Hierarchy window, right-click and select Add Source. Search for the file system_pn_top.vhd in your project directory and click Open. Associate it with All and click OK.
    3. In the Design window, select the Simulation view radio button and select Behavioral or Post-Route from drop-down menu.
    4. Edit the system_pn_top.vhd file. If it is in a non-editable state, close it and reopen it by double-clicking on the file in the Sources window. Modify the code in the system_pn_top.vhd file as shown below and save the file.
      library IEEE;

use IEEE.STD_LOGIC_1164.ALL;

entity system_pn_top is

       port (fpga_0_clk_1_sys_clk_pin : in std_logic;

             fpga_0_rst_1_sys_rst_pin : in std_logic );

end system_pn_top;

architecture STRUCTURE of system_pn_top is

       component system_xps is

            port (	fpga_0_clk_1_sys_clk_pin : in std_logic;
                   fpga_0_rst_1_sys_rst_pin : in std_logic );

       end component;

       attribute box_type : string;

       attribute box_type of system_xps : component is "user_black_box";

       begin

           system_xps_i: system_xps PORT MAP(
		
       fpga_0_clk_1_sys_clk_pin => fpga_0_clk_1_sys_clk_pin,
       fpga_0_rst_1_sys_rst_pin => fpga_0_rst_1_sys_rst_pin );

end architecture STRUCTURE;
    5. Highlight the system_pn right-click and select New Source. Create a new file named tb.vhd. Associate the new source with system_pn_top module.
    6. Edit the tb.vhd file
      LIBRARY ieee;

USE ieee.std_logic_1164.ALL;

USE ieee.std_logic_unsigned.all;

USE ieee.numeric_std.ALL;

ENTITY tb IS

       port (fpga_0_clk_1_sys_clk_pin : in std_logic;
             fpga_0_rst_1_sys_rst_pin : in std_logic );

END tb;

ARCHITECTURE behavior OF tb IS
 
-- Component Declaration for the Unit Under Test (UUT)

COMPONENT system_pn_top

               PORT(fpga_0_clk_1_sys_clk_pin : IN  std_logic;

                    fpga_0_rst_1_sys_rst_pin : IN  std_logic);

END COMPONENT;

BEGIN

-- Instantiate the Unit Under Test (UUT)

       uut: system_pn_top PORT MAP (fpga_0_clk_1_sys_clk_pin =>
       fpga_0_clk_1_sys_clk_pin,

       fpga_0_rst_1_sys_rst_pin => fpga_0_rst_1_sys_rst_pin);

END ARCHITECTURE behavior;
    7. While selecting the tb.vhd file in the Sources window, expand the simulator tree in the Processes window. .
    8. Double-click Simulate Behavioral or Post-Place & Route Model.
  10. This step is specific to cases 5 and 6.
    1. Add system_xps.ucf file to the project. This file is located in the data directory. Select the XPS design. Switch to the Simulation View and select Behavioral. Click Generate HDL Test Bench (XPS).
      The test bench is generated and attached to the design.
    2. Select the test bench and double-click Simulate Behavioral Model to run simulation.
      The simulator is launched and completes simulation.
  11. This step is specific to cases 7 and 8.
    1. Add system_xps.ucf file to the project. This file is located in the data directory. Select the XPS design.
    2. Switch to the Simulation View and select Post-Route.
    3. Click Generate HDL Test Bench.
      The test bench is generated and attached to the design.
    4. Select the design and run Simulate Post-Route Simulation Model.
