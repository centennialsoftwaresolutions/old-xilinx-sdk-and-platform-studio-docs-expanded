AXI Interconnect Configuration Dialog Box
The AXI Interconnect Configuration dialog box provides an easy way to configure and view the AXI Interconnect Parameters for all connected Masters and Slaves.
Using the AXI Interconnect Dialog Box
  1. The simplest way to use the AXI Interconnect dialog box is choosing any top level page and configuring the parameters in that page or in its sub pages.
  2. Click OK to close the dialog box and apply the changes.
Note For more information about AXI Interconnect, see the AXI Interconnect Reference Guide.
The following are the top-level pages in the AXI Interconnect Configuration dialog box.
Interconnect Strategy Tab
In the Interconnect Strategy tab, select a predefined interconnect strategy to view details about that strategy. You can select one of the following options. Click an option in the dialog box to view more details about that configuration type.
  • Current Settings - Apply this configuration to have your project retain the current settings. Use this option to cancel other applied strategies.
  • Minimum Area - Apply this configuration to implement the smallest possible interconnect core.
  • Maximum Performance - Apply this configuration to support high burst data throughput.
  • Typical (Restore Defaults) - This configuration consists of the default interconnect strategy, which is a suitable starting point for selectively optimizing performance of pathways to meet system throughput requirements.
To apply a selected strategy, click Apply. Your selections in this tab are not applied to your project until you click OK in the dialog box.
General Tab
  • Shows number of connected masters and slaves along with the width of all ID signals for all Masters, Width of all ADDR signals for all masters and slaves.
  • Provides appropriate entries boxes to configure the range check, interconnect data width, and user signal widths for all masters and slaves.
Control Interface Tab
  • Option to enable or disable AXI control interface.
  • Provides appropriate entry boxes for Interconnect Base and High addresses, control port protocol, address and data width.
Master/Slave Specific Settings
This page shows the parameters for all connected masters and slaves. A maximum of 16 masters and slaves can be connected to an AXI Interconnect IP.
The left side displays a list of parameter categories; use this to navigate to subpages where you can set parameters and values for masters and slaves.
The parameter and value set is shown in a parameter table. Each row in the table indicates one master or slave and first column shows name of master or slave as <IP_INST_NAME>.<BUS_IF_NAME>.
Note In each column in the table, you can copy parameter values from one master to all masters, or from one slave to all slaves. To do this, right-click on a value to copy and select CopyDown. The remaining rows are populated with the selected value.
  • Master ACLK & Other Settings Clock settings of each Master ACLK. Includes the following subpages:
    • Master Name - The name of the master.
    • ACLK Async - Allows you to specify whether the clock for each master is asynchronous or synchronous. This is an optional update parameter. If you provide a selection for one of the Masters, you have to provide a choice for all Masters.
    • ACLK Ratio - Displays the clock frequency ratio for each Master/Slave.
    • Arbitration Priority - The priority for the clock in arbitration.
    • Base ID - Settings for the Unique ID of the Master. You can choose to provide base ID for NO master or ALL masters.
    • Thread ID Width - Shows the thread ID width
    • Single Thread - Select whether to make this a single thread or not.
  • Master Register Slices — Settings for each channel, one sub page for each channel. Includes the following subpages:
    • Master - Master name.
    • Read Address - Register Slice Insertion method for the Read Address Channel for each Master.
    • Read Data - Register Slice Insertion method for the Read Data Channel for each Master.
    • Write Address - Register Slice Insertion method for the Write Address Channel for each Master.
    • Write Data - Register Slice Insertion method for the Write Data Channel for each Master.
    • Write Response - Register Slice Insertion method for the Write Data Channel for each Master.
  • Master Read/Write Settings — Settings for Master Read and Write Transactions. Includes the following subpages:
    • Master - Master name.
    • Read FIFO Depth - Depth of Read Data FIFO near the Bus Interface for each Master.
    • Write FIFO Depth - Depth of Write Data FIFO near the Bus Interface for each Master.
    • Read FIFO Delay - Delay issuing ARVALID until the read data FIFO has enough vacancy to store the entire burst length.
    • Write FIFO Delay - Delay issuing AWVALID until the complete burst is stored in the write data FIFO.
    • Read Issuing - Settings for the maximum number of active read transactions that the Master can generate.
    • Write Issuing - Settings for the maximum number of active write transactions that the Master can generate.
  • Master Information — Shows information about all connected Masters. Includes the following subpages:
    • Master - The name of the Master.
    • Slot - The slot number of the Master.
    • Data Width - The data width of the Master.
    • Protocol - The protocol of the Master.
    • Supports Read - Whether or not the Master has read support.
    • Supports Write - Whether or not the Master has write support.
    • Supports Narrow Burst - Whether or not the Master has narrow burst support.
  • Slave ACLK & Other Settings — Clock settings of each Slave ACLK. Includes the following subpages:
    • Slave - Slave name.
    • ACLK Async - Allows you to specify whether the clock for each Slave is asynchronous or synchronous. This is an optional update parameter. If you provide a selection for one of the Slaves, you have to provide a choice for all Slaves.
    • ACLK Ratio - Displays the clock frequency ratio for each Master/Slave.
    • Secure - Setting for the requirement of TrustZone security on all accesses of each connected slave.
  • Slave Register Slices — Settings for each channel, one sub page for each channel. Includes the following subpages:
    • Slave - Slave name.
    • Read Address - Register Slice Insertion method for the Read Address Channel for each Slave.
    • Read Data - Register Slice Insertion method for the Read Data Channel for each Slave.
    • Write Address - Register Slice Insertion method for the Write Address Channel for each Slave.
    • Write Data - Register Slice Insertion method for the Write Data Channel for each Slave.
    • Write Response - Register Slice Insertion method for the Write Data Channel for each Slave.
  • Slave Read/Write Settings — Settings for Slave Read and Write Transactions. Includes the following subpages:
    • Slave - Master name.
    • Read FIFO Depth - Depth of Read Data FIFO near the Bus Interface for each Slave.
    • Write FIFO Depth - Depth of Write Data FIFO near the Bus Interface for each Slave.
    • Read FIFO Delay - Delay issuing ARVALID until the read data FIFO has enough vacancy to store the entire burst length.
    • Write FIFO Delay - Delay issuing AWVALID until the complete burst is stored in the write data FIFO.
    • Read Acceptance - Settings for the maximum number of active read transactions that the Slave can accept.
    • Write Acceptance - Settings for the maximum number of active write transactions that the Slave can accept.
  • Slave Information — Shows information about all connected Slave. Includes the following subpages:
    • Slave - The name of the Slave.
    • Slot - The slot number of the Slave.
    • Data Width - The data width of the Slave.
    • Protocol - The protocol of the Slave.
    • Supports Read - Whether or not the Slave has read support.
    • Supports Write - Whether or not the Slave has write support.
    • Supports Narrow Burst - Whether or not the Slave has narrow burst support.
Interconnect Settings for BUSIF
This tab provides appropriate entry boxes for configuring S_AXI_CTRL bus Interface settings. Configuration of this bus interface is not supported.
Buses
  • This tab shows an option to enable disable CTRL port. This is disabled and unchangeable.
