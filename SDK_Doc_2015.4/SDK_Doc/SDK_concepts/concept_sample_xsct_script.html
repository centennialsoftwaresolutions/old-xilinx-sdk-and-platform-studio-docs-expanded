Sample XSCT Script

The script below provides a usage example of XSCT. This script creates and builds an application, connects to a remote hw_server, initializes the Zynq PS connected to remote host, downloads and executes the application on the target. These commands can be either scripted or run interactively.
Note: The workspace created in XSCT can be used from SDK. However, at a time, only one instance of the tool can use the workspace.
   # Set SDK workspace
sdk set_workspace /tmp/workspace
# Create a HW project
sdk create_hw_project -name hw1 -hwspec /tmp/system.hdf
# Create a BSP project
sdk create_bsp_project -name bsp1 -hwproject hw1 -proc ps7_cortexa9_0 -os standalone
# Create application project
sdk create_app_project -name hello -hwproject hw1 -bsp bsp1 -proc ps7_cortexa9_0 -os standalone 
-lang C -app {Hello World}
# Build all projects
sdk build_project
# Connect to a remote hw_server
xsdb connect -host raptor-host
# Select a target
xsdb targets -set -nocase -filter {name =~ “ARM* #0}
# System Reset
xsdb rst -system
# PS7 initialization
namespace eval xsdb {source /tmp/workspace/hw1/ps7_init.tcl; ps7_init}
# Download the elf
xsdb dow /tmp/workspace/hello/Debug/hello.elf
# Insert a breakpoint @ main
xsdb bpadd -addr  &main
# Continue execution until the target is suspended
xsdb con -block -timeout 500
# Print the target registers
puts [xsdb rrd]
# Resume the target
xsdb con
