Creating a Boot Image

Creating a boot image requires the following steps:

  1. Make sure that the FSBL project and any C application project are created in the SDK workspace and built so that corresponding ELF files are available.
  2. Create a new BIF file, select the path for creating a BIF file. The output path at the bottom is automatically populated with same path as that of BIF file path with the output image name as output.bin
  3. Enable authentication and select PPK, SPK PSK and SSK values.
  4. Enable encryption and select the key file and key source.
    Note: SPK signature is not required here, as you will provide the secret key PSK.


  5. Add the partitions one at a time, starting from the bootloader.
    1. Select the Partition type as “bootloader”.
    2. Select the FSBL “zynq_fsbl.elf” path.
    3. Select the authentication as rsa and encryption as aes.
  6. Click OK.


    Note: Presign is not required here, as we are providing the secret key SSK.
  7. Click OK.

    Notice that the list in Boot Image Partitions is populated.



  8. Add the second partition. For this example, you will add a C application.
    1. Select the Partition type as datafile.
    2. Select the application zynq_hello.elf path.
    3. Select the authentication as rsa and encryption as aes.
  9. Click OK.
    Note: Presign is not required here, as we are providing the secret key SSK.


  10. Click OK.

    Notice that the list in Boot Image Partitions is updated with the second partition.



  11. Click Create Image to create the final boot image “output.bin”.
