xsdbserver start

Description

Start XSDB command server.

Usage

 xsdbserver start [Options]

Start XSDB command server listener. XSDB command server allows external processes to connect to XSDB to evaluate commands. The XSDB server reads commands from from the connected socket one line at the time.

After evaluation a line is sent back starting with "okay " or "error " followed by the result or error as a backslash quoted string.

Options

 -host <addr>

Limits the network interface on which to listen for incoming connections.
-port <port>

Specified port to listen on. If this option is not specified or if the port is zero then a dynamically allocated port number is used.
