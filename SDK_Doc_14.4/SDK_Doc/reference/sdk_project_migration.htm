Migrating SDK 12.x projects to SDK 14.x and Higher

Migrating from SDK 12.x to SDK 14.x and higher versions should be seamless in most cases.

There is one manual step involved if your projects use Cygwin style paths. In such cases, these paths should be converted to use Windows style paths or relative paths. SDK 13.x does not include Cygwin, and the GNU toolchain used is built on MingW.

Migrating SDK 11.x projects to SDK 12.x

To migrate SDK 11.x projects to 12.x, please follow these steps.

Create a New Hardware Platform Project in SDK 12.x

  1. Open your EDK design in XPS 12.1
  2. Click through the version migration wizard.
  3. In XPS, select Project > Export Hardware Design to SDK
  4. Select the Export & Launch SDK button

SDK should now open with a hardware platform project that corresponds to your hardware.

Import Existing BSP

  1. Click on File -> Import
  2. Expand "General" and select Existing Projects into Workspace.
  3. Navigate to the 11.x workspace and find your Software Platform / BSP from the previous project and select OK
  4. Select the software platform project to be imported.
  5. Enable the "Copy Projects into Workspace" option, and press Finish.
  6. After the project is imported, a pop-up message will indicate that the project should be closed and re-opened.
  7. Right Click on the newly imported BSP and select Close Project.
  8. Right click on the BSP project again and select Open Project.

To make sure that the BSP project has been updated properly, right click on the BSP and click on "Board Support Package Settings". Make sure the dialog comes up properly. If it fails to come up, that implies that the BSP project has not been imported properly. In such a case, please create a new BSP using SDK 12.x and manually copy over the MSS file into the new BSP and name it system.mss (overwriting the existing file).

Import Existing C Application Project

  1. Click on File > Import
  2. Expand "General" and select Existing Projects into Workspace.
  3. Navigate to the 11.x workspace and find your C application project from the 11.x project and select OK
  4. Select the software project to be imported.
  5. Enable the "Copy Projects into Workspace" option, and press Finish.
  6. Let SDK migrate the build settings from the 3.0.0 format to the 4.0.0 format

If you get an error after importing your 11.x C application project, select Project > Clean after which the project will automatically rebuild and clean out any inconsistent data files from the 11.x build to the 12.x project.
