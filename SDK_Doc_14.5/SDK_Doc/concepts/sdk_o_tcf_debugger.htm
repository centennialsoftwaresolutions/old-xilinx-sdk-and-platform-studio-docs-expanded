Debugging with TCF

This document provides the details that are needed for a user to start using the new customized TCF debugger integrated into Xilinx SDK.

The TCF debugger from open source is customized and is integrated with Xilinx SDK.

Apart from the Xilinx Customized TCF debugger, the XMD/GDB debug flow that is supported in earlier versions of the tool can still be used. Users will see an improved performance in this release due to the updates in XMD and underneath cable infrastructure to support coalescing of multiple JTAG commands and the usage of TCF agent for JTAG communication.

The TCF debugger support is integrated only for ARM over Digilent cable. Users can use the SDK-XMD flow for the Xilinx USB cable which will continue to work as released in the earlier versions. The scope of this document will be focused towards the new customized TCF debugger over Digilent cable for ARM. Following sections describe installation, setting up the environment and debugging applications using the TCF integrated SDK.
