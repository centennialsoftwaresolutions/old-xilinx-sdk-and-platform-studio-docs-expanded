Known Issues

The following are known issues in SDK:

  • XPPU configuration requirment for system debugger
  • No IP instance named psu_pcie_high present in hardware design error in SDK log
  • While debugging the BSP code, some variable values are displayed as N/A
  • Exporting project to GIT server causes problems in BSP/Application Compilation
  • Projects created in the default location may cause compilation issues
  • After installing SDK on Ubuntu machines, toolchain paths are not accessible from SDK
  • Ucos application project creations fail due to non-availability of pre-compiled libraries and source code
  • Debugging or running of A53 32-bit application requires board reset
  • MSS Viewer does not refresh automatically
  • Renaming hardware platforms causes compilation errors
  • Restore defaults in C/C++ Build Settings causes settings to be lost
  • SDK does not display correct menus, icons or perspective
  • Debug System Reset doesn’t work on a 2.0. Zc706 board that has the 2.0 silicon
  • SDK Hangs After Design Changes in Block Design and Export Hardware (Design with User IP)
  • While Initializing the bitstream with an ELF file, SDK uses bootloop
