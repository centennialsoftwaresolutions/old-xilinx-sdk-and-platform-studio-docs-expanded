Known issues in SDK

Memory Viewer displays data values in the wrong endian format

  • Issue: The memory viewer displays data in big endian format.
  • Unfortunately, there are no solutions for this issue right now. You must manually parse the memory contents in the correct format.

While Initializing the bitstream with an ELF file, SDK uses bootloop

  • Issue: The Configure FPGA dialog allows you to select the ELF file to be initialized in the bitstream. When this dialog box opens, it remembers the ELF file used in a previous invocation. However, regardless of the ELF shown in the dialog box, SDK always uses bootloop to initialize the bitstream.
  • Solution: In the dialog box, manually reselect the ELF to use.

MSS Viewer does not refresh automatically

  • Issue: The MSS viewer does not automatically refresh when either the MSS is changed via the BSP settings dialog box, or when the hardware specification is updated.
  • Solution: Close the MSS viewer and then double click on the MSS file to re-open it.

Program FPGA dialog does not appear

  • Symptom: The Program FPGA dialog does not appear. Clicking on the Program FPGA toolbar item or the menu item does not result in any action.
  • Possible Issue: This issue could arise if any 11.x projects were imported, but the import process was not successful. Make sure that the application project's referenced projects includes the board support package (BSP), and that the BSP's referenced project includes the hardware specification project.

Renaming hardware platforms causes compilation errors

  • Symptom: When a hardware specification project is renamed, the dependent BSPs and applications fail to compile.
  • Issue: Currently, SDK does not support renaming of hardware specification projects. In order to do so, you must manually change the following items:
    1. Change Project References in the BSP project to point to the new hardware project.
    2. Change the libgen.options file to specify the correct hardware specification file.
    3. Change the ELFCheck options in the application's build settings.

Restore defaults in C/C++ Build Settings causes settings to be lost

  • Issue: Clicking on Restore Defaults in the C/C++ Build Settings results in elfcheck not working properly.
  • Solution: Manually set the path to the hardware specification and the processor name in the ELFCheck options in Build settings after restoring defaults.

Operations on XMD console can confuse SDK's internal state

  • Currently, SDK uses the XMD that is shown in the XMD console. As a result, performing some operations on the XMD console (such as disconnecting targets) will cause SDK's state to get out of sync with the actual target state. In such a case, it might not be possible to recover and you must restart SDK.

When SDK launches, it doesn't display the correct menus, the correct icons, or the C/C++ Perspective

  • Solution: SDK is probably reading settings from an outdated .eclipse folder or an outdated workspace. Try opening SDK with a fresh workspace. If that does not work, remove your $HOME/.eclipse folder. (On NT, this is usually C:\Documents and settings\username\.eclipse.)

Kintex™-7 FPGA KC705 evaluation board with older device revisions require a workaround during debugging to compensate for the memory controller calibration time

  • Symptom: After any reset, the debugger hangs when accessing DDR3 memory.
  • Issue: For older device revisions, the memory controller requires one to two minutes to calibrate before you can access DDR3 memory. To work around this issue, allow sufficient time for calibration to complete after a reset and disable automatic resets using one of the following methods:
    1. When using XMD, after connecting to the target, disable resets running the commands debugconfig -reset_on_run disable and debugconfig -reset_on_data_dow disable.
    2. When using SDK, in the debug configuration Device Initialization tab specify the Reset Type value No Reset. Use XMD to perform the reset and wait for the calibration to complete.
  • See also: Answer Record 43967 contains additional information.
