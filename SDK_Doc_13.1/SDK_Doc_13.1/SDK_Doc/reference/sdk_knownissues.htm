Known issues in SDK

Target pattern contains %

Symptom: Project clean/build/auto rebuild etc. complain with the error message that "Target pattern contains %". This issue occurs on Windows only.

Possible Issue: This situation can arise when a user manually enters a Windows style path to the CDT build system. e.g.: If user adds the path "C:\path\to\library" as an additional include folder. The issue arises because of conflicts between Cygwin make's syntax rules and the Windows style paths that have a drive name (C:). In such a case, make assumes that C is a target to be built.

Solution: Users should be careful not to use any Windows style paths. Ideally, relative paths should be used whenever any paths are specified. If relative paths are not an option, use a Unix style path suitable for Cygwin.

In the above case, it should be "/cygdrive/c/path/to/library".

Program FPGA dialog does not show up

Symptom: The Program FPGA dialog does not show up. Clicking on the Program FPGA toolbar item, or the menu item does not result in any action.

Possible Issue: This issue could potentially arise if any 11.x projects were imported, but the import process was not successful. Please make sure that the application project's referenced projects includes the BSP, and the BSP's referenced project includes the hardware specification project.

Renaming h/w platforms

Symptom: When a hardware specification project is renamed, the dependent BSP's and applications fail to compile.

Issue: Currently, SDK does not support renaming of hardware specification projects. If such a need arises for the user, then the user has to manually change the following items:

  • Change Project References in the BSP project to point to the new hardware project.
  • Change the libgen.options file to specify the correct hardware specification file.
  • Change the ELFCheck options in the application's build setting

Restore defaults in C/C++ settings causes settings to be lost.

Issue: Clicking on Restore Defaults in the C/C++ Build Settings will result in elfcheck not working properly.

Solution: Manually set the path to the hardware specification and the processor name in the ELFCheck options in Build settings after restoring defaults.

Operations on XMD console can confuse SDK's internal state

Information: Currently, SDK uses the same XMD as shown in the XMD console. As a result, performing some operations on the XMD console (such as disconnecting targets) will cause SDK's state to get out of sync with the actual target state. In such a case, it may not be possible to recover and SDK will have to be restarted.

When SDK opens up, it doesn't have the right menus, or the right icons, or the C/C++ perspective

Solution: SDK is probably reading settings from an outdated .eclipse folder, or an outdated workspace. Try opening SDK with a fresh workspace. If that doesn't work, remove your $HOME/.eclipse folder. (On NT, this is usually C:\Documents and settings\username\.eclipse)

When a BSP project is deleted, SDK warns you if dependent C/C++ projects exist, but the deletion cannot be cancelled

Solution: This is a known issue. You must associate the C/C++ projects with a compatible BSP project. In the Project Explorer tab, right-click on the application project and select Change Referenced BSP. Then select the BSP and click OK.

SDK video does not play on Windows 64-bit platforms when the Watch Now button is clicked on the Welcome page

Solution: This is a known issue. The browser might appear on Windows 64-bit platforms, but the video does not play.
