Creating a New Zynq FSBL Application Project

To create a new Zynq® FSBL application in SDK, do the following:

  1. Click File > New > Application Project.
    Note:     This is equivalent to clicking on File > New > Project to open the New Project wizard, selecting Xilinx > Application Project, and clicking Next.
     The New Application Project dialog box appears.
  1. In the Project Name field, type a name for the new project.
  2. Select the location for the project. To use the default location as displayed in the Location field, leave the Use default location check box selected. Otherwise, click to unselect the check box, then type or browse to the directory location.
    The Hardware Platform drop-down list comes with pre-defined hardware platforms for ZC702 , ZC706 & ZED hardware. You can use this to start your application on the fly, instead of going for any hardware design creation tools. These are specific to the PS-only part of Zynq.
  3. Select the options for the target hardware:
    1. Select the appropriate target platform from the Hardware Platform drop-down list.
    2. Select the Create New option from the list to create a new hardware platform project.
    3. From the Processor drop-down lists, select the appropriate target processor.
  4. Select the options for the target software:
    1. From the OS Platform drop-down list, select the target OS Platform.
    2. Select the appropriate Language option. For this example, select C/C++.
    3. Select the Create New option of Board Support Project to create a new board support package project.
      Note: Alternately, select Use existing to select the existing Board Support Package projects.
  5. Click Next.
  6. In the Templates dialog box, select the Zynq FSBL template.
  1. Click Finish to create your application project and board support package (if it does not exist).
