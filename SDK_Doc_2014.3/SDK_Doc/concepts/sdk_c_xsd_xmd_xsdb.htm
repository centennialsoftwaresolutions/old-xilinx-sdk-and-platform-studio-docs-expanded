XMD vs XSDB

This section lists some of the differences between XMD and XSDB commands from the usage perspective.

connect

In both XMD and XSDB, connect is the primary command to establish connections to debug targets. However, there is a difference in how connect works in XMD and XSDB.

XMD provides the connect command to connect to a debug target, such as a processor target.

In XSDB, the connect command opens a connection to the hw_server. Use the targets command to list all the available targets and select a debug target before issuing any debug commands to that target.

targets

In XMD, the target IDs have fixed values (For example, MB target IDs start with 0 and Cortex-A9 target IDs start with 64).

In XSDB, the target IDs are sequential. Targets are assigned IDs in the order they are detected on the scan chain. If the scan chain is partial during connect command, XSDB assigns IDs to new targets when they become visible.

To select a target programmatically, XSDB provides the filter option to be used with the targets command. Below is an example of the use of filter .

targets -filter {name =~ "ARM*#1"} -set

Ctrl+C

With XMD, Ctrl+C terminates the application immediately.

In XSDB, you can use Ctrl+C to terminate long running commands like program FPGA or download a file. Press Ctrl+C twice in succession to terminate XSDB.

Program FPGA

In XSDB, the fpga command can be used to program a FPGA. However, unlike XMD, when using XSDB you must use the targets command to select the FPGA device before running the fpga command.

Running Scripts

Scripts can be sourced from the XSDB terminal or specified while launching XSDB.

When the script is specified during launch, arguments to the script can follow the script name. XMD exits after running the script; XSDB doesn't exit.
