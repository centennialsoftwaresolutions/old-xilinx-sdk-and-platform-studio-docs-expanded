Xilinx System Debugger Overview

This document is intended to provide usage help for the Xilinx System Debugger CLI (XSDB).

XSDB is created to give a user-friendly, interactive, and scriptable command line interface to Xilinx hw_server and other incarnations of TCF servers used by Xilinx. A major source of inspiration for the commands in XSDB comes from Xilinx Microprocessor Debugger (XMD).

Differences between the two are designed to do the following:

  • Allow interaction with the whole system
  • Support a software engineer's view of both hardened and programmable logic
  • Provide performance measurement
  • Integrate with hw_server and other incarnations of TCF servers

As with XMD, the XSDB scripting language is based on the Tools Command Language (Tcl).
