SDK Headless Mode

SDK headless mode can be used to create, import and build projects using the command line.

Assumptions

The variables used in the steps below are as follows:

  • XILINX_SDK_PATH: SDK installation Path (C:/Xilinx/SDK/2014.1).
  • ECLIPSE: a shortcut to the Eclipse console application inside SDK installation:
    $( XILINX_SDK_PATH)/eclipse/win64.o/eclipsec.exe, or $(XILINX_SDK_PATH)/eclipse/lnx64.o/eclipse depending on the platform.
  • WSPACE: path to the workspace to be used.
  • VM: Path to Java VM used by SDK. This should be: $(XILINX_SDK_PATH)/tps/win64/jre/bin

The steps also reference the following Eclipse/SDK projects, all of which are typically stored in the same folder.

  • HW: path to the SDK hardware platform project
  • BSP: path to the SDK BSP project
  • BSPNAME: SDK BSP project name
  • APP: path to SDK C/C++ application project
  • APPNAME: SDK C/C++ application project name

User has to source the settings file in the shell/command prompt depending on the platform.

$(XILINX_SDK_PATH)/settings64.bat (or) $(XILINX_SDK_PATH)/settings64.csh

Creating Projects

User can create below mentioned project types in headless mode.

  • Hardware Platform Project
  • Board Support Package Project
  • Application Project

To create projects in headless mode, a script file need to be given as input for SDK.

Syntax:

xsdk -script <script.xml> -workspace <workspace directory path>

This script file(script.xml) contains information about the projects to be created. The script file can contain one or multiple tasks to create one of the project types.

<project name="SDK Script" default="main">

<target name="main">

<createHwProject ...

<createBspProject ...

<createAppProject ...

<createAppProject ...

<createAppProject ...

</target>

</project>

Each script file must have project and target xml tags as specified above. In addition to these mandatory tags, there are three tasks (each specified with an xml tag) which can be specified in the file. It is allowed to have multiple tasks of a type in a single script. Given below is the description of the tasks.

createHwProject

This task is used to create a Hardware Platform project. Parameters that you must specify for the task are:

  • projname: Name of the Hardware Platform project to be created in the workspace.
  • hwspecpath: Path of the design specification file exported from Vivado/XPS.
Example

<createHwProject projname="zynq706_two_mb" hwspecpath="C:\Designs\ zynq706_two_mb\system.xml " />

createBspProject

This task is used to create a Board Support Package project. Parameters that you must specify for the task are:

Parameter

Description

Example

projname

Name of the Board Support Package project to be created in the workspace.

 

hwprojname

Name of the hardware platform project existing in the workspace.

 

processor

Name of the processor supported by the hardware platform.

ps7_cortexa9_0 , microblaze_0

os

Name of the operating system supported by the processor.

standalone , linux , xilkernel

Example

<createBspProject projname="zynq706_two_mb_bsp1" hwprojname="zynq706_two_mb" processor=" ps7_cortexa9_0" os="standalone" />

createAppProject

This task is used to create an Application project. Parameters that you must specify for the task are:

Parameter

Description

Example

projname

Name of the Application project to be created in the workspace.

 

hwprojname

Name of the hardware platform existing in workspace or one of the predefined templates.

MicroZed_hw_platform, ZC702_hw_platform, ZC706_hw_platform, zed_hw_platform

processor

Name of the processor supported by hardware platform.

ps7_cortexa9_0, microblaze_0

os

Name of the operating system supported by the processor

standalone, linux, xilkernel

language

ID of the languages supported.

C, CPP

template

Name of the template supported for the configuration

Hello World, Empty Application

bspprojname

Name of the bsp project to be created or name of the valid existing bsp project.

 
Example

<createAppProject projname="zynq706_two_mb_perph_app" hwprojname="zynq706_two_mb" processor="ps7_cortexa9_0" bspprojname="zynq706_two_mb_bsp1" os="standalone" template="Peripheral Tests" language="C" />

Sample File Reference

xsdk -script <script.xml> -workspace <workspace directory path>

Refer to the sample script.xml file.

Importing Projects

To import the three projects into the workspace, issue the following command from a shell:

$(ECLIPSE) -vm $(VM) -nosplash -application org.eclipse.cdt.managedbuilder.core.headlessbuild -import $(HW) -import $(BSP) -import $(APP) -data $(WSPACE) -vmargs -Dorg.eclipse.cdt.core.console=org.eclipse.cdt.core.systemConsole

Note: If your workspace already has these three projects, then this step can be skipped.

Building Projects

To compile projects from command line, issue following commands based on your requirement.

Build All Projects:

$(ECLIPSE) -vm $(VM)-nosplash -application org.eclipse.cdt.managedbuilder.core.headlessbuild -build all -data $(WSPACE)-vmargs -Dorg.eclipse.cdt.core.console=org.eclipse.cdt.core.systemConsole

Build BSP:

$(ECLIPSE) -vm $(VM)-nosplash -application com.xilinx.sdk.sw.BspBuilder $(BSPNAME) -data $(WSPACE) -vmargs -Dorg.eclipse.cdt.core.console=org.eclipse.cdt.core.systemConsole

Build Application:

$(ECLIPSE) -vm $(VM)-nosplash -application org.eclipse.cdt.managedbuilder.core.headlessbuild -build $(APPNAME) -data $(WSPACE)-vmargs -Dorg.eclipse.cdt.core.console=org.eclipse.cdt.core.systemConsole
