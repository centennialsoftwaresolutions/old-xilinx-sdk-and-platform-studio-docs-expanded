Implementing Your Design in XPS
To implement your design in XPS, you must first set the implementation options.
Note If the XPS project (Xilinx® Microprocessor Project (XMP) file) has been instantiated as a source in a Project Navigator project, these settings are not required and are unavailable.
To set the implementation options:
  1.  Open the Project Options dialog box for a project by selecting Project > Project Options.
  2.  Select the Hierarchy and Flow tab of the Project Options dialog box.
  3.  Click to select the Implement Design in XPS (XPS Flows) check box.
    Note You can only select the Implement Design in XPS option when the embedded module represents the complete FPGA design and there are no other components. You can use this option to implement the design using Xflow or Xplorer.
  4.  In the Effort Level to Run FPGA Implementation Tools drop-down list, select one of the following options:
    •  Xflow (single iteration)
      Xflow allows control over FPGA implementation tool options with the fast_runtime.opt file.
    •  Xplorer Scripts (multiple iterations for best result)
      Xplorer automatically figures out the required options needed for implementation tools to achieve the desired result, such as meeting timing constraints. Xplorer runs multiple iterations of the implementation tools and uses the results of one run to determine the implementation tool options for the next run. It continues until the desired result is obtained. Since Xplorer uses multiple iterations, it might require longer runtime. At the end of Xplorer, the implementation tool options that resulted in the desired results are copied into etc/xplorer_fast_runtime.opt. To reduce runtime for future runs, you can copy the best options as determined by Xplorer into fast_runtime.opt and use those options with Xflow. Non-trivial changes to the design might require a change in the options necessary for implementation tools, so you might need to re-run Xplorer if the design has changed and implementation tools are not achieving the desired results.
  5.  Click to select the Treat timing closure failure as an error check box to have XPS treat the inability of place-and-route (PAR) tools to meet timing constraints as an error instead of a warning. This stops the implementation flow and a bitstream is not generated. This functionality is new in the 9.2i release.
  6.  Click to select the Improve timing on high-fanout nets check box to enable tools to optimize the high-fanout reset nets. Use this option if implementation tools cannot meet timing constraints on reset nets in your design.
    This setting is deprecated and will be removed in the EDK 11.1 release. For equivalent functionality, use the -register_duplication switch directly in the ISE™ MAP tool when generating your bitstream.
    Note  For this switch to be effective, the MAP tool must be run with the register_duplication switch. When you select Xflow to implement your FPGA design, XPS automatically adds the correct map option. However, when you use Xplorer or when using Project Navigator to implement your design, you must ensure that the appropriate map options are selected.
Starting with 10.1, Xilinx ISE™ tools will perform optimizations on all nets above a certain fanout threshold (not just reset net) when the -register_duplication switch is used with MAP. However, if the fanout of the reset net does not fall above the ISE threshold, you can still use the switch to force tools to do optimization on the reset net. The high fanout threshhold is determined internally by the MAP tool and cannot be changed.
The mechanism to export your XPS design to an ISE project was deprecated in EDK 8.1 and was removed for EDK 10.1. To implement your design using ISE, Xilinx recommends that the XPS module be instantiated as a source in Project Navigator. To use the export flow, run xps -nw and use the "run exporttopn" command. This functionality is not available in XPS.
