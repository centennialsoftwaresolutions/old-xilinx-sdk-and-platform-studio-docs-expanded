Organization of Generated Files
The Create and Import Peripheral wizard generates files based on your input. The following table describes what files are generated and how they are used.
Directory or File Description
<pcores-directory>One of the following: <EDK-Repository-Dir>/MyProcessorIPLib/pcores or <Directory-containing-XPS-Project-File>/pcores
<logical-library-name>The logical library name.
<peripheral-name>The peripheral name.
<peripheral-version>The peripheral version.
<peripheral-directory><pcores-directory>/<logical-library-name>
<devl>peripheral-directory>/devl A directory containing collateral files to help you develop the user-logic component of the peripheral.
<devl>/README.txtA file explaining the output generated by the wizard. Xilinx recommends that you look through this file. It has a lot of documentation about exactly what you must do to complete the implementation of the user-logic part of the peripheral.
<devl>/ipwiz.logA file containing a list of messages outputted by the wizard.
<devl>/ipwiz.optA file capturing the data input in the wizard. You do not need to use this file for any purpose.
<projnav-dir><devl>/projnav A directory containing a Xilinx® ISE™ Project Navigator project file. This directory contains files used by Project Navigator if you choose to develop the user-logic part of the peripheral using Project Navigator.
<projnav-dir>/<peripheral-name>.iseThe Project Navigator project file you can open to complete the development of the peripheral.
<projnav-dir>/<peripheral-name>.cliDo not use this file. It is used by the wizard to generate the Project Navigator project file.
<synthesis-dir><devl>/synthesis A directory containing files that help you synthesize the peripheral using XST.
<synthesis-dir>/<peripheral-name>_xst.prjThe XST project file. If you add more Hardware Description Language (HDL) files to your peripheral, you must add them to this file.
<synthesis-dir>/<peripheral-name>_xst.scrA simple XST script file that uses the XST project file and can be passed to XST to generate the netlist representing the peripheral.
<simulation-dir><devl>/bfmsim A directory containing files that help you simulate the design using Bus Functional Model (BFM).
<simulation-dir>/README.txtA README file that contains instructions on how to start BFM simulation.
<simulation-dir>/bfm_sim_cmd.makeA makefile for command line usage only. It contains targets for BFM simulation platform compilation and launching simulator.
<simulation-dir>/bfm_sim_xps.makeA makefile for XPS only. It contains targets for BFM simulation platform compilation and launching a simulator.
<simulation-dir>/bfm_system.mhsA BFM simulation system testbench description file, input to Simgen for behavioral simulation generation.
<simulation-dir>/bfm_system.mssAn empty system driver file to work around XPS warnings.
<simulation-dir>/bfm_system.xmpAn XPS project file for BFM simulation only.
<simulation-dir>/scripts/sample.bflCoreConnect bus transactions described in Bus Functional Language that are input to the Bus Functional Compiler to generate simulator commands for simulation.
<simulation-dir>/scripts/wave.doA signal dataset file for viewing all signals in the waveform window.
<simulation-dir>/scripts/run.doThe top-level simulator script file. Contains commands to compile, load modules, and start simulation.
<ip-testbench-name><peripheral-name>_tb
<ip-testbench-dir><simulation-dir>/pcores/<ip-testbench-name>_<peripheral-version> A directory containing the IP testbench file for the peripheral.
<ip-testbench-dir>/hdl/vhdl/<ip-testbench-name>.vhdAn IP testbench file. Defines the processes used to test the peripheral and provides constant interface to the system testbench and mechanism to communicate with Bus Functional Language commands for synchronization.
<ip-testbench-dir>/data/A directory containing EDK interface files (Microprocessor Peripheral Description (MPD) and Peripheral Analyzer Order (PAO) files) for the IP testbench.
<peripheral-directory>/dataA directory containing EDK interface files (MPD and PAO) for the peripheral.
<peripheral-directory>/hdl/vhdlA directory containing generated or imported VHDL files representing the peripheral. If you need more VHDL files to represent your peripheral, you can add them here.
<peripheral-directory>/hdl/verilogA directory containing generated or imported Verilog files representing the peripheral. If you need more VHDL files to represent your peripheral, you can add them here.
<drivers-directory>One of the following: <EDK-Repository-Dir>/MyProcessorIPLib/drivers or <Directory-containing-XPS-Project-File>/drivers
<peripheral-driver-directory><drivers-directory>/<logical-library-name>
<peripheral-driver-directory>/dataA directory containing EDK interface files (Microprocessor Debug Definition (MDD) and Tcl files) for the peripheral driver.
<peripheral-driver-directory>/srcA directory containing peripheral driver source files and the makefile for compilation.
