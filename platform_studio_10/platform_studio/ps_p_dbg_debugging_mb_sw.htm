Debugging MicroBlaze Software
To debug MicroBlaze™ software, do the following.
  1.  Start the Xilinx® Microprocessor Debugger (XMD) and connect to the MicroBlaze processor. To do this:
    In XPS, select Debug > Launch XMD.
    Now XMD is connected to MicroBlaze and has started the GNU debugger (GDB) server port 1234 for source level debugging using the GDB console.
  2.  Start the STDIN/OUT Terminal in XMD. To do this:
    1.  In the XMD shell, type terminal -jtaguart_server. This opens a terminal TCP/IP server. The port number is listed on the XMD shell.
    2.  Use a Hyperterminal application, such as Win HyperTerminal or TeraTerm, and make a TCP/IP connection to this server.
    Note This does not apply to XMDStub target communicating over a serial interface.
  3.  Start the GDB (mb-gdb) Console. To do this:
    1.  In XPS, select Debug > Launch Software Debugger.
    2.  Connect GDB to the XMD GDB server TCP port 1234 and debug the program locally (hostname=localhost) or remotely (hostname = IP addr).
    3.  For help in using GDB, select Help > Help Topics in GDB.
For more information, refer to:
  •  The MicroBlaze Processor Reference Manual
  •  The Xilinx Microprocessor Debugger (XMD) chapter in the Embedded System Tools Reference Manual
