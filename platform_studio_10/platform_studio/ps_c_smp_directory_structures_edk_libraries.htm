Directory Structures for EDK Libraries
Xilinx® EDK tools handle the aspects of embedded design. For any such design element to be recognized by EDK tools, it must be present in a certain directory, and it must have a pre-defined structure. Each design element has a meta-data file associated with it, which provides information about that design element to the EDK tools.
Some of the design elements and the meta-data files associated with them are listed below.
Design ElementPSF FileDirectory Within Library
Hardware IPsMicroprocessor Peripheral Definition (MPD) filepcores
Device DriversMicroprocessor Driver Description (MDD) filedrivers
Software LibrariesMicroprocessor Library Definition (MLD) filesw_services
Prototype BoardsXilinx Board Definition (XBD) fileboards
Board Support PackageMicroProcessor Library Definition
(MLD) file		bsp
EDK supports the concept of a library. A library can contain any of the above design elements. However, EDK tools search for a given design element type only in a particular directory within the library. Those directory names are listed in the table. For example, EDK tools search for hardware IPs only in the pcores directory.
You can specify a library search path in the XPS Project Options menu. This path should not include the library directory itself. Each directory within the library search path is assumed to be a library.
You can organize your design elements in multiple libraries but you must still specify only one directory path which contains all the libraries.
XPS displays a dialog box if you add a search path directory that does not have any pcores, sw_services, or drivers subdirectories. The dialog displays information similar to the following:
"<dir> does not seem to be a valid search path\n"
"A Valid search path must have a subdirectory that has at least a pcores subdirectory\n"
"For example, if your IP is at C:\\EDK_Libs\\MyRepository\\pcores\\myip_v1_00_a\n"
"then, the search path should be C:\\EDK_Libs\n"
Note Xilinx IPs are also organized in the same manner. For example, the %XILINX_EDK%/hw directory is one such library path. It contains multiple libraries, such as XilinxProcessorIPLib, XilinxBFMInterface, and XilinxReferenceDesigns.
EDK tools automatically search libraries within the installation area. Aside from these, tools also search for libraries in the directory edk_user_repository, present at the same level as EDK installation directory for other libraries. You can install third-party design elements in this directory rather than explicitly specifying the search path in each project.
In the Application Preferences page of the Preferences dialog box, specify the Global Repository Search Path to apply to all projects. This is the recommended method for adding search paths that apply to all projects.
