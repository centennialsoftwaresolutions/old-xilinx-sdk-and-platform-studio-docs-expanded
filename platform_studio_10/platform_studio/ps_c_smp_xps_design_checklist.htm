XPS Design Checklist
This page provides a summary of all necessary steps and commonly used optional steps to complete an embedded processor system design.
An asterisk (*) shows that a step can be automated or partially automated when you create a basic system with the Base System Builder (BSB) and target a supported development board.
Hardware Development Tasks
  •  Create or open an XPS project as described in Starting a New Project. You can create a project using one of the following two methods:
    •  Create a project with the Base System Builder (BSB) as described in  Base System Builder.
    •  Create a project from scratch within XPS as described in Creating a Blank XPS Project.
  •  Specify the target FPGA device as described in Setting Device and Repository Options.*
  •  Add processor and peripheral cores to the design as described in Adding IP Modules to the Hardware Platform.*
  •  Connect peripherals via processor buses as described in Making Bus Connections Among IP Modules.*
  •  Design and connect custom peripherals as described in Using the Create and Import Peripheral Wizard (optional).
  •  Interconnect non-bus ports of IP cores as described in Making Port Connections Among IP Modules.*
  •  Define all external ports at the embedded system interface as described in Making Port Connections Among IP Modules.*
  •  Connect interrupts from peripherals through an interrupt controller to the processor as described in Connecting Interrupts (optional).*
  •  Specify the system address map for bus peripherals as described in Generating the System Address Map.*
  •  Specify the configuration parameters for each of the processor or peripheral cores as described in Customizing IP Modules.*
  •  Specify the FPGA pin assignment in the User Constraints File (UCF) file as described in Implementing the Hardware Platform (usually).*
  •  Specify timing constraints (at least the processor reference clock period) as described in Implementing the Hardware Platform.*
  •  Configure the download.cmd file to match the JTAG chain on the board as described in Specifying the JTAG Chain (usually).*
  •  Add the embedded system as a submodule to a top-level Xilinx® ISE™ project in Project Navigator; declare, instantiate, and interconnect the embedded submodule in your top-level FPGA design (optional). This is described in Using ISE Project Navigator to Implement Your FPGA Design.
  •  Add and connect Xilinx ChipScope™ cores to your design to aid in hardware debugging as described in Debugging Hardware Using ChipScope Pro (optional).
  •  Develop simulation scripts and testbenches as described in Introduction to Simulation in Platform Studio (optional).
  •  Generate Xilinx System ACE™ files to perform FPGA configuration as described in Generating a System ACE File (optional).
Software Development Tasks
  •  Specify software platform settings (OS, drivers, and libraries) as described in Assigning Drivers, Libraries, and Operating Systems. This can also be done in Platform Studio SDK.
  •  Create a software project to include in the FPGA bitstream as described in Creating a New Software Application Project Using XPS (usually).
  •  Select a startup application, OS, bootloop, or bootloader to include in the FPGA bitstream as described in Initializing Software Overview.
  •  Write and debug software application code as described in Creating Software Application Projects Overview.
  •  Include headers for Board Support Package (BSP) (xparameters.h), drivers, and libraries in your source code as described in Generating the Software Libraries. This can also be done in Platform Studio SDK.
  •  Write drivers for custom peripherals as described in Using the Create and Import Peripheral Wizard (optional).
  •  Write or generate a linker script as described in Generating Linker Scripts (optional).
  •  Set compiler and linker options as described in Setting Compiler Optimizations (usually).
  •  Prepare software images and program off-chip application storage, typically flash memories as described in Programming Flash Memory (optional). This can also be done in Platform Studio SDK.
