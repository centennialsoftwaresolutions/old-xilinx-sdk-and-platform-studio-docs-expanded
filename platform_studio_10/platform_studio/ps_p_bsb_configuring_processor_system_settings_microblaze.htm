Configuring Processor and System Settings for MicroBlaze in the Base System Builder
Based on the processor selected, you can configure system and processor-specific settings.
System settings include processor and bus clock frequencies. Allowable values might be restricted by the clock resources available on the target development board or the on-chip resources available in the FPGA device.
Processor-specific settings include debug interfaces, cache options, and configuration of any on-chip memory that communicates over a processor-specific bus.
To Configure Processor and System Settings for a MicroBlaze System
  1.  Specify a reference clock frequency.
    The reference clock is the external clock source on the board you are using. If you selected the Custom Board option or if the board you are using has a configurable clock source (such as a socket for an oscillator), the reference clock frequency is editable. You must ensure that the value you enter here is the frequency of the actual clock source on your board.
  2.  Select a processor-bus clock frequency.
    The BSB offers a selection of frequencies that can be generated from the reference clock source. The values allowed might be dependent on the FPGA you are using because certain on-chip resources such as Digital Clock Managers (DCMs) might be required to perform clock division or multiply. In MicroBlaze™ systems, the processor and buses must all run on the same clock frequency.
  3.  Select a debug interface.
    Use the Debug I/F radio buttons to show how you intend to debug software running on the processor:
    •  On-chip H/W debug module - Select this if you use Xilinx® Microprocessor Debugger (XMD) in conjunction with a hardware debug module (mdm) implemented on the FPGA fabric.
    •  XMD with S/W debug stub - Select this if you use XMD in conjunction with a debug module implemented in software.
    •  No debug - Select this if you do not use any software debugger.
  4.  Select local data and instruction memory.
    Note The selection of memory sizes depends on the chosen FPGA architecture. This drop-down box shows the size of on-chip data and instruction memory accessed through the Local Memory Bus (LMB). The LMB is designed to allow fast memory access, so it is best not to overload the LMB by allocating a large amount of local memory. If your application is too big to fit in local data and instruction memory, you can use more block RAM or external memory through memory controllers on the bus. In such cases, using cache (described below) enhances system performance.
  5.  Decide whether to enable caching. To enable caching, select the Enable check box.
    Note MicroBlaze has a cache option that enables applications to execute faster. You can select from the following cache options:
    •  No Cache - Disables the MicroBlaze cache.
    •  Enable cache - Enables the MicroBlaze cache feature. Caching is performed through a dedicated transparent FSL interface between the MicroBlaze cache and a multi-channel memory controller. You are required to add only one multi-channel memory controller to your system if you select this option. This CacheLink option is only available if the board you selected contains a memory device for which a multi-channel controller exists.
  6.  Enable the floating point unit.
    Enabling the floating point unit feature allows you to accelerate the arithmetic using real numbers.
  7.  Click Next.
