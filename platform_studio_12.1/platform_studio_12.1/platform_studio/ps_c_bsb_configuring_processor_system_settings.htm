Configuring Processor and System Settings in the Base System Builder
In the Processor Configuration page of the Base System Builder (BSB), configure the processor(s) in your design. One or two sets of configuration options are available, depending on the system type you selected (single-processor or dual-processor) and the board(s) selected for your design.
To Set the Reference Clock Frequency and Select the Processor Type(s)
  1. In the Reference Clock Frequency field, type the frequency (in MHz).
    The reference clock is the external clock source on the board you are using. If you selected the Custom Board option or if the board you are using has a configurable clock source (such as a socket for an oscillator), the reference clock frequency is editable. You must ensure that the value you enter here is the frequency of the actual clock source on your board.
  2. In the Processor Type drop-down list, select your processor type (MicroBlaze or PowerPC).
    Note If you selected a dual-processor system, the BSB provides two sets of configuration options.
To Set Configuration Options for MicroBlaze Processors
When you select a MicroBlaze™ processor type, you can configure the following details for the processor.
  1. In the System Clock Frequency drop-down list, select the frequency for the system clock, in MHz.
    The BSB offers a selection of frequencies that can be generated from the reference clock source. The values allowed might be dependent on the FPGA you are using because certain on-chip resources such as Digital Clock Managers (DCMs) might be required to perform clock division or multiply. In MicroBlaze processor systems, the processor and buses must all run on the same clock frequency.
  2. In the Local Memory drop-down list, select the amount of local memory, in KB.
    Note The selection of memory sizes depends on the chosen FPGA architecture. This drop-down box shows the size of on-chip data and instruction memory accessed through the Local Memory Bus (LMB). The LMB is designed to allow fast memory access, so it is best not to overload the LMB by allocating a large amount of local memory. If your application is too big to fit in local data and instruction memory, you can use more block RAM or external memory through memory controllers on the bus. In such cases, using cache (described below) enhances system performance.
  3. In the Debug Interface drop-down list, On-Chip HW Debug Module is selected by default, which specifies that the design uses Xilinx® Microprocessor Debugger (XMD) in conjunction with a hardware debug module (mdm) implemented on the FPGA fabric. This cannot be changed.
To Set Configuration Options for PowerPC Processors
When you select a PowerPC® processor type, you can configure the following details for the processor.
  1. In the Processor Clock Frequency drop-down list, select the frequency for the processor clock, in MHz.
    BSB offers a selection of frequencies that can be generated from the reference clock source. The values allowed might be dependent on the FPGA you are using because certain on-chip resources such as Digital Clock Managers (DCMs) might be required to perform clock division or multiplication.
  2. In the Bus Clock Frequency drop-down list, select the frequency for the bus clock, in MHz.
  3. In the On-chip Memory drop-down list, select the amount of on-chip memory (OCM), in KB.
    The selection of memory sizes depends on the chosen FPGA architecture. The OCM controller serves as a dedicated interface between the block RAMs in the FPGA and OCM signals available on the embedded PowerPC 405 peripheral. The OCM signals on the processor block are designed to provide quick access to a fixed amount of instruction and data memory. The OCM controller, an integral component of the PowerPC 405 peripheral architecture, provides an interface to both the 64-bit Instruction-side block RAM and the 32-bit Data-Side block RAM.
  4. In the Debug Interface drop-down list, FPGA JTAG is selected by default, which specifies that the design uses the on-chip JTAGPPC module to debug software. This cannot be changed.
  5. Click to select the Enable Floating Point Unit check box if you want to enable the Floating Point Unit (FPU) feature.
    Note The FPU feature is only supported if the bus clock integer ration for bus:PPC is from 1:1 to 1:16 and the bus clock is less than 225 MHz.
After setting the configuration options for your processor(s), click Next to continue to the Peripheral Configuration page of the BSB.
