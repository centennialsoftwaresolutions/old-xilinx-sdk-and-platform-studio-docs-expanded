What’s New in SDK

Updates

The following are updates in SDK 2014.1:

  • The SDK interface has a new look, including a new Welcome screen with helpful links.
  • The System debugger has been updated for Cross-Trigger support for Zynq®-7000 devices.
  • Download speed for Program FPGA and ELF images is improved.
  • The GCC toolchain for MicroBlaze™ processors is now updated to 4.8.3.
  • SDK is now controlled by the license manager. To run SDK, you must use either a 30-day evaluation mode or a permanent license.
  • SDK now only supports 7 series and above devices, as well as Zynq devices.

SDK no longer supports 6 series and earlier devices. For support for these devices, use SDK version 2013.4.

  • PowerPC support is now deprecated.
  • Libgen support is now deprecated. A new infrastructure is implemented in SDK. Use Headless mode for command line support.
New Features

SDK 2014.1 includes the following new features:

  • New Target Connection Feature is implemented for any connections to the board. This includes the following additional changes:
    • The Configure JTAG Settings feature is deprecated.
    • All features such as Program FPGA and Debug Launch now include the Target Connection.
  • SDK log is now shown in a separate view.
  • A new Performance Analysis feature is implemented. You can use this to display ARM performance and AXI performance in both tabular and visual modes.
  • New Regenerate BSP Sources feature allows you to regenerate the BSP on demand. Previously the BSP was automatically generated; now you can choose to regenerate it.

The following features are in Alpha mode this release:

  • A new mode, called Headless mode, allows you to create SDK workspace projects from the command line. Refer to SDK Headless Mode.
  • System Debugger Tcl Mode (XSDB). Refer to System Debugger Tcl Mode (XSDB).

 

For changes in earlier releases, refer to What’s New in SDK 2013.4 and Earlier Releases.
