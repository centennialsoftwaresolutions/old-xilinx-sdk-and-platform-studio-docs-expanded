Dubugging a Remote ARM Linux Application

To debug a remote ARM Linux application, first open the Run Configurations dialog box by selecting Run > Run Configurations, or by right-clicking on your application and selecting Run > Run Configurations.

Do the following in the Main tab:

  1. In the Project field, type the name of the project containing the application that you want to run, or browse to it.
  2. In the C/C++ Application field, type the name of the executable that you want to run. You can also search for the file or browse to it.
  3. In the connection field, select New > General > SSH Only and click Next.
    1. In the New connection window, type the Target board IP in the Host name field
    2. Type the connection name and description in the respective fields.
    3. Click Finish to create the connection.
    4. The Connection field now shows the Target IP.
  4. In the Remote Absolute File Path for C/C++ Application field, type the path in Linux Target that contains the location of the Application to be loaded and run.
  5. In the Commands to execute before application field, type the commands that need to be executed before running the application.
    For example, as shown in the figure below, the application FastCorner_sdk.elf uses OpenCV libraries Path, which is set in this field.
  6. Click Apply to save the settings.
  7. Click Run.
