Generating the System Addresses Automatically
The Address Generation utility in XPS can be used to automatically assign addresses. To generate new addresses in your system, click Generate Addresses at the top of the System Assembly View. XPS automatically generates all address ranges in your hardware platform.
The Address Generation Utility computes the base addresses and high addresses of peripherals in processor-based designs. The addressing rules for PowerPC® and MicroBlaze™ processor-based systems are complex. This utility helps to obviate that complexity by reducing the address specification process to a few mouse clicks. In addition, it is able to accommodate some common OS requirements. The Address Generation utility employs a set of heuristics to determine the address maps for all the peripherals. These heuristics account for the following:
  • For MicroBlaze processor systems, the utility assumes that the memory connected through IXCL and DXCL bus interfaces is cached.
  • The PowerPC reset address is 0xFFFFFFFC and the MicroBlaze reset address is 0x0. This utility ensures memory peripherals at this address. For multiple memory peripherals, the largest on-chip memory gets the reset address.
  • Many operating systems require some addressable memory at 0x0, so this utility ensures that the largest off-chip memory is at that address for PowerPC systems. If that is not possible, some memory peripheral is at that address.
  • When there is only one memory peripheral, the reset address requirement takes precedence.
  • When there are no memory cores in the system, other addresses are generated 1 Mb away from the reset address.
The Address Generation Utility does not generate address maps for certain peripherals, such as the following:
  • If a peripheral’s bus interface is not connected to the processor (through bus and sometimes bridges), all address parameters for the core are left unchanged.
  • Some address parameters are controlled by some other parameters. For example, PLB_V46 has the following parameter definitions in its Microprocessor Peripheral Definition (MPD) file: 
    PARAMETER C_BASEADDR = 0b1111111111, DT = std_logic_vector, BUS = SDCR, 
    MIN_SIZE = 0x08, ASSIGNMENT = REQUIRE, ISVALID = (C_DCR_INTFCE==1) PARAMETER 
    C_HIGHADDR = 0b0000000000, DT = std_logic_vector, ASSIGNMENT = REQUIRE, ISVALID = (C_DCR_INTFCE==1) 
    PARAMETER C_DCR_INTFCE = 0, DT = integer, BUS = SDCR, RANGE = (0,1)
    When C_DCR_INTFCE = 0, the registers addressed by C_BASEADDR and C_HIGHADDR in this peripheral are not accessible from the processor, so the address parameters are left unchanged.
  • By default, the address generator tool preserves the current address map for register type of addresses.
    To have the address generator re-generate the address, do the following:
    • Change the base address to 0.
    • Select the size.
    • Generate the address.
    Note The address generator tool might not preserve the address map for memory peripheral addresses.
  • For bridges, all address range parameters are generated. Other parameters (such as C_NUM_ADDR_RNG, which determines how many ranges are used) are also generated.
  • For MicroBlaze processors, if any peripheral is cached, the following parameters are generated, based on the generated instruction and data cache ranges:
    • C_ICACHE_BASEADDR
    • C_ICACHE_HIGHADDR
    • C_USE_ICACHE
    • C_ADDR_TAG_BITS
    • C_DCACHE_BASEADDR
    • C_DCACHE_HIGHADDR
    • C_USE_DCACHE
    • C_DCACHE_ADDR_TAG
For the address already present in the design, the tool will try to preserve this address. In some cases, however, it might override those addresses. These address values are checked for compliance against the following common addressing rules:
  • The base address must be lower than the high address.
  • The size of address space implied by the base address and high address must be a power of two.
  • The base address must be on the 2n bit boundary. If any of these conditions is not met, this utility exits with an error and leaves the design unchanged.
