Instantiating and Connecting the Embedded Submodule
ISE® provides a Hardware Description Language (HDL) instantiation template file that represents a top-level design containing the embedded submodule. You can use this template to copy component declaration and instantiation sample into your top-level HDL design.
Note This step is only necessary if the embedded module is part of a larger design.
  1. In Project Navigator, select your embedded processor source in the Design Hierarchy pane.
  2. In the Processes pane, run View HDL Instantiation Template to open the template in the Project Navigator editor pane.
  3. Copy the component declaration for the embedded system (VHDL) and paste it into your top-level design architecture.
  4. Copy the instantiation sample of the embedded system into your top-level design and provide net name connections as necessary.
You can connect output ports of your embedded submodule to output ports and to other loads in your top-level design. You can drive input ports of your embedded design from input ports or other logic in your top-level design. The component port interface of the embedded submodule has a one-to-one correspondence with the External Ports in the MHS file.
Alternatively, you can run the new Project Navigator process Generate Top HDL Source to generate a simple HDL wrapper for your XPS design. This enables you to get started with Project Navigator quickly, and without having any additional design logic in place other than you XPS design.
  1. In Project Navigator, select your embedded processor source in the Design Hierarchy pane.
  2. In the Process pane, run Generate Top HDL Source.
    The generated top is added to your project automatically, and set as the design top.
Note Implementation processes are no longer available when you select your embedded processor source, even if it set as the design top.
Starting with the 11.1 version of the tools, Project Navigator can automatically include constraints that are associated with the embedded submodule into the flow. By default, these are the constraints located in data/system.ucf in your EDK project directory.
Note Your embedded design can no longer be implemented as the top module in Project Navigator. You must instead reference from some higher level module, using either the instantiation template method, or the top resulting from running the new Generate Top HDL Source process in Project Navigator.
In order to enable automatic inclusion of embedded constraints, you must mark the embedded submodule as a black box in the top-level HDL. This can be done by adding the box_type attribute with the value user_black_box. Project Navigator copies the synthesized netlist for the EDK module (by default, located in implementation/system.ngc) into its own project directory. It also copies the data/system.ucf file from EDK to the system.ncf file in its individual project directory. This process enables the FPGA Implementation Tools (process Translate, ngdbuild) to use the synthesized netlist for the EDK module and its associated constraints.
Note Designs from previous versions of tools (such as 10.1) might not have the EDK submodule marked as a black box in the top-level HDL file. In this case, the system.ncf file is not read-in by the FPGA Implementation Tools; you must manually add the constraints associated with the embedded submodule into your top-level User Constraints File (UCF) in Project Navigator.
