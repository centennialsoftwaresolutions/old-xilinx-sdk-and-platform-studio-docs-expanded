AXI Auto Bus Connections
Making Auto Bus Connections
When adding an IP instance to an AXI design, XPS looks up the address map of a MicroBlaze™ processor to identify the best existing IP instance as a reference, and using the reference to connect AXI Bus Interface of the new IP instance. In addition, XPS also assigns address, connect clock and make IO ports external for the new IP instance. When there are multiple MicroBlaze™ processors, you can choose which processor to connect to from an XPS pop-up dialog. From the same dialog, you can choose to skip automation, or add in the IP instance without any automation. When there is no MicroBlaze in the design, XPS tries to find a reference IP instance in the design and make the connection. If for any reason XPS cannot make the connection automatically, XPS adds in the new IP instance without automation. In this case, you have to connect bus interface, assign address, connect clock and make IO ports external manually. This functionality is for incremental design updates, assuming an initial working design has been created from the Base System Builder. XPS prints messages to the console to inform you about what changes were made, and/or what changes must be done manually.
Note The automation in this release is for AXI register slave, AXI memory slave, MicroBlaze and DMA-style IPs. For DMA-style IPs, XPS connects the master interface to the main memory of MicroBlaze system, and connects the slave interface similarly as for an AXI register slave IP. XPS determines the register or memory by looking at the ADDR_TYPE tag in the address parameters associated with the AXI slave interfaces.
The automation goes through the following steps:
  1. Bus Interface Connections and instantiation of related IP’s
    • XPS only looks at valid bus interfaces at the time of instantiation of the IP
    • XPS adds related IP’s when necessary. For example, when you add in an AXI bram controller, XPS automatically adds in a bram_block IP instance and connects the BRAM_PORTA and BRAM_PORTB of the axi_bram_ctrl instance to the PORTA and PORTB of the bram_block instance.
  2. Clock Connections
    • For the new IP instance which is connected to an axi_interconnect, XPS looks for the signal connected to the axi_interconnect instance INTERCONNECT_ACLK port, and connect the same signal to the AXI bus interface’s clock of the new IP instance.
    • You need the Clock Wizard to ensure that all clocking requirements are complete. The auto-bus connections do not generate necessary clocks or make all the necessary clock connections.
  3. IO Connections
    • Make all valid ports which have IO_IS tags to be external.
  4. Address Generation
    • Run address generator utility to generate address.
      Note If you want to keep a specific address range from being changed by Address Generator, please lock the address in Addresses tab before adding any IP instance.
MicroBlaze Automation
The automation for MicroBlaze has the following steps:
  1. Select local bram size.
    • • In the pop up dialog, select the size of local memory in the new MicroBlaze. You can select to not use any local memory.
  2. Create lmb_bram memory, set local memory to the size from step 1, and make necessary connections.
    • • This step is skipped if you choose to not use local memory in step 1.
  3. Create a new axi_interconnect instance and connect it to the peripheral (M_AXI_DP) interface.
  4. Connect instruction cache (M_AXI_IC) and data cache (M_AXI_DC) interfaces to memory.
  5. Connect the debug interface to MDM.
  6. Connect reset ports of the new MicroBlaze instance and related IPs to the proc_sys_reset instance.
  7. From the reference MicroBlaze, find the slow clock from the axi_interconnect, which is connected to the peripheral (M_AXI_DP) interface, and find the fast clock from the axi_interconnect, which is connected to instruction cache (M_AXI_IC) and data cache(M_AXI_DC) interfaces. In the new MicroBlaze, connect the slow clock to the clock port of the axi_interconnect, which is connected to peripheral (M_AXI_DP) interface; connect the fast clock to the clock port of the new MicroBlaze instance. If the new MicroBlaze has local memories, connect their clock ports to the fast clock.
  8. The remaining steps are the same as for AXI resister slave and AXI memory slave IPs.
