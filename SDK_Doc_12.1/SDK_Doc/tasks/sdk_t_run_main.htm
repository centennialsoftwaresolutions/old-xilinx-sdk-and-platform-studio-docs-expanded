Selecting an application to run/debug/profile

You must specify the project or application program that you want to run, debug, or profile for this run configuration.

To select an application to run

  1. In the C/C++ Projects view, select a project.
  2. Select Run > Run Configurations.
  3. In the Configurations list of the window that opens, expand Xilinx C/C++ ELF and select a run configuration.
  4. In the Name field, type a descriptive name for the launch configuration.
  5. In the Main tab, do the following:
    • In the Project field, type the name of the project containing the application that you want to run, or browse to it.
    • In the C/C++ Application field, type the name of the executable that you want to run. You can also search for the file or browse to it.
  6. Do the following, as required:
    • Specify the initialization tasks that should be done before starting debugging (for information, see Specifying initialization tasks).
    • Use the SDK console as STDIO for your application (for information, see Specifying STDIO Connection).
    • Specify the options for profiling (for information, see Specifying Profiler configuration).
    • Specify the options for debug agent XMD (for information, see Specifying XMD target configuration).
  7. Click Run.

To select an application to debug

  1. In the C/C++ Projects view, select a project.
  2. Select Run > Debug Configurations.
  3. In the Configurations list of the window that opens, expand Xilinx C/C++ ELF and select a debug configuration.
  4. In the Name box, type a descriptive name for this new a descriptive name for this launch configuration.
  5. In the Main tab, do the following:
    • In the Project field, type the name of the project containing the application that you want to run, or browse to it.
    • In the C/C++ Application field, type the name of the executable that you want to run. You can also search for the file or browse to it.
  6. Click Debug or do the following, as required:
    • Specify the initialization tasks that should be done before starting debugging (for information, see Specifying initialization tasks).
    • Use the SDK console as STDIO for your application (for information, see Specifying STDIO Connection).
    • Specify the options for profiling (for information, see Specifying Profiler configuration).
    • Specify the options for debug agent XMD (for information, see Specifying XMD target configuration).
